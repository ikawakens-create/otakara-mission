# SPEC.md — 機能仕様

「おたからミッション」アプリの機能仕様。画面・ロジックの振る舞いを定義する。
データ構造の詳細は `DATA_MODEL.md` を参照。

---

## 1. 全体像

子どもが毎日ミッションにスタンプを押す → 1日全クリアでボーナス＋ガチャ →
景品を集めてアバターを着せ替え＆スタンプ図鑑を埋める → 週5日クリアで週間ボーナス。
おうちの人は別モードで設定・換金管理する。

---

## 2. 画面一覧

### 2.1 プロフィール選択 / ホーム
- 起動時、姉／妹のどちらかを選ぶ（前回選択を記憶、上部でいつでも切替可）。
- ホームに表示するもの：
  - 自分のアバター（現在の着せ替え状態）
  - 名前、今日の日付・曜日（自動）
  - 今日のポイント・累計ポイント・かけら数
  - 今日のミッション一覧（各ミッションにスタンプ枠）
  - 「全部できた！」状態のときはコンプリート表示＋ガチャボタン

### 2.2 ミッション＆スタンプ押し
- ミッションをタップ → **スタンプ選択パレット**が開く。
- 所持スタンプから好きなものを選んで「ペタッ」と押す（押下時：弾むアニメ＋効果音）。
- 押し間違いは**長押しで取り消し**（誤操作防止のため単タップでは消えない）。
- スタンプ済みミッションは見た目で分かる（押されたスタンプを表示）。
- 「じぶんできめたミッション」は子どもが内容を選択/入力できる枠を1つ用意。

### 2.3 ガチャ
- 1日コンプリート時に **1回**引ける（その日まだ引いていなければ）。
- 演出：レバーを回す/カプセルが落ちる等のワクワク演出 → レア度に応じた光・音・エフェクト。
- 結果表示：出た景品（ポイント/スタンプ/アイテム）とレア度。ダブりなら「かけら +N」表示。
- レア度別の演出差を必ずつける（ダイヤモンドは専用ファンファーレ）。詳細は §4。

### 2.4 コレクション
- **スタンプ図鑑**：全スタンプの一覧。未所持はシルエット表示（集める動機）。
- **アイテム図鑑**：着せ替えアイテムをカテゴリ別に一覧。未所持はシルエット。
- 取得率（例：32/120）を表示。

### 2.5 アバター着せ替え
- 自分の女の子キャラに、所持アイテムを着せる。
- カテゴリ：かみがた / ふく / ぼうし / アクセサリー / はいけい / ペット（拡張可）。
- 選んで即プレビュー、保存。ホームのアバターに反映。

### 2.6 こうかんじょ（かけら交換所）
- かけらを使って、欲しいアイテム/スタンプと交換。
- 価格はレア度に応じる（DATA_MODEL §かけら参照）。
- ダブりで腐らせず「最終的に狙ったものが手に入る」救済。

### 2.7 おうちの人モード（PINロック）
- 4桁PINで保護。子どもが誤って入れない場所に管理機能を分離。
- できること：
  - ミッション項目の編集（追加/削除/並べ替え/絵文字変更）
  - ポイント→円の換算レート設定、週上限の設定
  - 「今週のおこづかい精算」確認（§5）
  - データの書き出し/読み込み（バックアップ）
  - データリセット（確認ダイアログ必須）

## 3. ごほうびロジック

### 3.1 1日コンプリート
- その日の**全ミッション**にスタンプが押されたら「コンプリート」。
- コンプリート時に付与：**ボーナスポイント +1** ＆ **ガチャ1回**。
- スペシャル花丸演出（画面いっぱいの花丸＋紙吹雪＋効果音）。
- 同日中の二重付与を防ぐ（`completedDate` / `gachaPulledDate` を記録）。

### 3.2 週間ボーナス
- 週は **月曜〜日曜**。
- その週で **5日以上**コンプリートしたら週間ボーナス（演出＋特別報酬：ボーナスポイントや限定スタンプ等）。
- 週の達成状況を示す進捗メーター（0〜7日）をホームか専用枠に表示。
- 週の境界をまたいだら自動でリセット（前週の達成数は精算後に繰り越さない）。

### 3.3 ポイント計算
- スタンプ1個 = 1ポイント（基本）。
- 「10じまでにねる」など指定ミッションは **2倍**（`doublePoints: true`）。
- コンプリートボーナス +1、週間ボーナス、ガチャ景品のポイントも加算。
- 累計ポイントと「今週分」を別々に集計（精算用）。

## 4. ガチャ仕様

### 4.1 レア度テーブル（合計100%）

| ランク | キー | 確率 |
|---|---|---|
| ノーマル | `normal` | 40% |
| レア | `rare` | 25% |
| スーパーレア | `super_rare` | 15% |
| 激レア | `ultra_rare` | 10% |
| 虹色 | `rainbow` | 5% |
| 超超超レア | `legend` | 3% |
| ダイヤモンド | `diamond` | 1% |

### 4.2 抽選の流れ
1. レア度テーブルで重み付き抽選し、ランクを決定。
2. そのランクの景品プールから1件を抽選。
3. 景品が「既所持のスタンプ/アイテム」なら**かけらに変換**して付与（ダブり救済）。
4. 景品が「ポイント」ならポイント加算。
5. 結果と演出をランク別に表示。

- ランクごとに演出（色・光・音・エフェクトの強さ）を変える。
- 景品プールが空のランクが出た場合のフォールバック（例：ポイント付与）を必ず用意し、エラーで止めない。

### 4.3 景品の種類
- **ポイント**：レア度が高いほど多い（例 normal +1 〜 diamond +10）。
- **スタンプ**：ミッションで使えるスタンプが増える。
- **着せ替えアイテム**：アバター用。コレクションの核。

具体的な確率値・付与量・プールは `DATA_MODEL.md` で数値として定義（コード分離）。

## 5. おこづかい精算（おうちの人モード）

- 週末（任意のタイミングで可）に「今週分ポイント × 換算レート」を表示。
- 例：今週 23ポイント × 10円 = 230円（上限設定があれば上限適用）。
- 精算したら「精算済み」にして今週分カウンタをリセット（累計は残す）。
- 換算レート・上限はおうちの人が設定。デフォルトは控えめ（例 1pt=10円, 週上限300円）。

---

## 6. 子ども向けUX要件

- 文字は最小限・ひらがな中心。重要操作は絵とアイコンで伝える。
- ボタンは大きく、押し心地（アニメ・音）を必ずつける。
- 取り消しは長押しなど誤操作しにくい操作に。
- ネットなしでも動く（オフライン前提）。
- 2人で見せ合えるよう、コレクションは「見て楽しい」レイアウトに。

---

## 7. 非機能要件

- オフライン動作（PWA・端末内保存）。
- データ消失を起こさない（書き込みは原子的に、スキーマにバージョン）。
- 端末スリープや再起動後も状態が残る。
- 追加コンテンツでコードを変更せずに済む（data配列追加のみ）。
