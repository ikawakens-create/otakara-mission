# AVATAR_SPEC.md — アバター着せ替えシステム 確定仕様

おたからミッションのアバター（着せ替え）システムの**正の仕様**。
このファイル単体で、いつでも・誰（別セッションのClaude／Claude Code）でも再開できることを目的とする。
※ここに書いた「確定」はチームの合意済み。「提案（次に確定）」「未確定」は別セクションに明記。

---

## 1. 確定した方式（コア）

### 1-1. レイヤー画像方式（手描きSVGは不採用）
- アバターは**透過PNG画像を重ねて**作る。コードで描くベクター/絵文字キャラは品質が足りないため不採用。
- 絵の供給源は **AI画像生成（Gemini）**。Claudeは「生成の指示書」「切り抜き」「実装」を担当（Claudeは絵を"描かない"）。

### 1-2. 素材パイプライン（並行作業の要）
```
Geminiで白背景(#FFFFFF)で生成
  → Claudeが背景を切り抜いて透過PNG化（+ 軽量化）
  → アプリへ（src の素材として配置＋データ1行追加）
```
- Geminiは透過が苦手なので**必ず白背景**で出す。透過はClaude側で行う。

### 1-3. 重ね順（z順）
**背景 → 着せた体（コーデ）→ 髪 → 帽子 → 小物 → ペット**
- 「着せた体（コーデ）」＝体＋顔＋服を1枚にした"はげの着せ替え済みボディ"。顔はこの層に含む。
- 「髪」は1枚でOK（顔の穴から体が見えるので前/後ろに分けない）。前髪あり/なしは"別の髪型"として持つ。

### 1-4. 基本立ち絵は2種＋特別立ち絵
- **基本立ち絵2種**：お姉ちゃん用・妹ちゃん用（別の顔・別の体）。**両方とも「正面・直立」**で固定（その形だから髪・服がピタッと合う）。
- **特別な立ち絵**：ポーズ＋服＋髪を**まるごと1枚**にした特別な姿（ピース・座り・ダンス・季節衣装など）。重ね合わせ不要＝位置合わせゼロで、どんなポーズもOK。
  - これは**ガチャの当たり景品／コンプリート報酬**にできる（既存ガチャと接続）。

### 1-5. 髪・服は姉用/妹用で別々に用意
- 2人は体（ポーズ）が別なので、**同じ服でも姉用・妹用を1回ずつ生成**する（姉の体に合わせた服は妹の体に合わない）。
- 服は"編集モード"（ベース画像を編集）で出せるので手間は軽め。

### 1-6. アバターをアプリ全体に登場させる
- ふだんは「いま着せ替え中のアバター」をホーム・ミッション・ガチャに小さく登場（同じ描画の使い回し＝追加コスト小）。
- 感情表現は**動き（CSS）＋キラ＋吹き出し＋特別立ち絵**で出す。
- **表情差分はやらない**（顔がベース立ち絵に含まれるため高コスト）。将来やるなら「顔を別レイヤー化」する別仕組みで（後回し）。

### 1-7. 軽量化
- 画像は枚数が増えると重いので、Claude側で実装時に**縮小・圧縮**する（井川さんは元画像を用意するだけ）。

## 2. 検証済みであること（実証ログ）
- 井川さんがGeminiで生成した「はげベース」「髪（穴あき）」「ピンクワンピ」「セーラー服」を、
  Claudeが切り抜いて〈着せた体→髪〉に重ねたところ、**1枚絵として成立**。髪×服の混ぜ合わせも破綻なし。
- **位置は0オフセットでほぼ一致**（同じベースに合わせて生成すれば自動で揃う）。微ズレはアプリの位置調整で対応可能。
- → レイヤー画像方式は「実現可能」と確認済み。

## 3. キャラ設定

### 3-1. 確定した雰囲気
- **妹ちゃん（いもうと）**：元気・にっこり全開・まるっと小さめでかわいい。テーマ色＝ピンク。背は少し低め。
- **お姉ちゃん（あね）**：すらっと・やさしい・おっとりした微笑み。テーマ色＝みずいろ／ラベンダー。背は少し高め。

### 3-2. 提案（次に確定）
- 髪型：妹＝黒〜こげ茶の**ボブ＋ぱっつん前髪** / 姉＝黒〜こげ茶の**ミディアム＋前髪**（着せ替えで変更自由）。
- 瞳の色：**こげ茶で揃える（自然）** を既定にする。ゲームっぽくするなら妹=ピンク瞳/姉=みずいろ瞳も可。← 要確定。

## 4. データ構造のスケッチ（実装用）

新コンテンツは `src/data/` に画像ベースで定義（コードと分離の原則は維持）。

- アバター素材レコード（例）：
  ```ts
  type AvatarAsset = {
    id: string;
    category: "outfit" | "hair" | "hat" | "accessory" | "pet" | "background" | "special";
    owner: "sister" | "younger" | "both";   // both は背景/ペット等の共用可
    label: string;          // ひらがな名
    imagePath: string;      // 透過PNG
    rarity?: Rarity;        // 図鑑/ガチャ連携用（任意）
    offsetX?: number;       // 位置微調整（既定0）
    offsetY?: number;
    scale?: number;         // 既定1
    starter?: boolean;      // 初期所持
  };
  ```
- AvatarConfig（profile内・既存を拡張）：
  ```ts
  type AvatarConfig = {
    outfitId?: string; hairId?: string; hatId?: string;
    accessoryId?: string; petId?: string; backgroundId?: string;
    specialId?: string;   // セット時は"特別立ち絵"をまるごと表示（レイヤー無視）
  };
  ```
- 所持は既存の `ownedItemIds` を流用（アバター素材のidを入れる）。

---

## 5. 描画・親の位置調整モード
- 描画：コンテナに z順で `<img>` を重ね、各画像に `offsetX/Y/scale` を適用（CSS transform）。
- `specialId` があれば、その1枚を表示（レイヤー描画はスキップ）。
- **親モードに「位置調整」**：選んだ素材の offset/scale をその場で微調整して保存。
  → Geminiの多少のズレを生成し直さずに直せる。確定値は最終的に data に焼き込む。

---

## 6. 他PR・既存仕様との関係（重要）
- **図鑑(3-A)と着せ替え(3-B)は同じ素材データを共有**する。
- 既存の絵文字ベースの `items.ts`（hair/clothes/hat… の絵文字）は**プレースホルダ**だった。
  アバターは画像レイヤー方式に移行するので、**`items.ts` を画像アバター素材に作り替える**。
  - → 図鑑は「絵文字」ではなく「アバター素材（コーデ/髪型…）」を並べる形に更新する。
  - 進行上、**アバター素材のデータ定義を先に置く**と、図鑑も着せ替えも同じ資産を読めて綺麗。

## 7. ビルド順（PRは小さく・1PR=1タスク）
1. **3-B 着せ替え**：レイヤー描画＋着せ替え画面＋ホームにアバター反映＋親の位置調整モード。
2. **アバター登場＆リアクション**：ホーム/ミッション/ガチャにアバター＋動き・キラ・吹き出し。
3. **特別立ち絵をガチャ景品に**：special をガチャ/コンプ報酬として組み込み、お披露目。
※ 図鑑(3-A)・バックアップUIは別途。図鑑は本仕様の素材データに合わせて実装/更新する。

## 8. 素材づくりタスク（Gemini 並行作業）

### 8-0. 共通ルール（各プロンプト先頭に付ける）
```
かわいいデフォルメ（ちび・約3頭身）、全身・正面・直立。
背景は純白(#FFFFFF)の無地。影・グラデ・模様・文字なし。
被写体は画像の端に触れないよう、まわりに白い余白を残す。
線画くっきり・アニメ塗り・縦長の画像。指定パーツ以外は描かない。
```

### 8-1. ベース（はげ・白背景）※まず妹→できたら妹を添付して姉
**妹ベース**
```
（↑共通ルール）
女の子を1人。頭は髪のない状態（つるつる）、白いノースリーブシャツと白い短パン、はだし。
顔：大きくまんまるの瞳（こげ茶）、ぷっくりほっぺ、口を開けたにっこり元気な笑顔。
雰囲気：元気いっぱい・まるっと小さめ。背は少し低め。髪は描かない。
```
**お姉ちゃんベース**（妹の完成画像を添付）
```
（↑共通ルール）添付の妹と同じ画風・塗り・線・頭身で"別の子（お姉ちゃん）"を描いて。
頭は髪のない状態（つるつる）、白いノースリーブシャツと白い短パン、はだし。
顔：大きめの瞳（こげ茶）、口を閉じたやさしい微笑み。
雰囲気：すらっとして少しお姉さんっぽい・やさしい・おっとり。背は妹より少し高め、手足ほっそりめ。髪は描かない。
```

### 8-2. 髪（白背景・パーツのみ）※対象のベースを添付し、姉用/妹用を別々に
```
（↑共通ルール）添付の子の頭に合う「髪」だけを描いて。○○（例：黒のボブ＋ぱっつん前髪／こげ茶のミディアム＋前髪）。
顔・体・手足は描かない。顔が来る中央はあける。髪色は○○。
```

### 8-3. コーデ（ベース画像を「編集」して出す）
※ベース画像を添付して「この画像を編集して」と指示
```
この子はそのままに、服だけ「○○（例：ふんわりピンクのフリルワンピース）」に変えて。
顔・表情・髪のない頭・ポーズ・体の位置・大きさは一切変えない。背景は純白(#FFFFFF)、影なし。
```

### 8-4. 特別立ち絵（まるごと1枚）
```
（↑共通ルール）添付の子（○○ちゃん）が、○○（例：両手バンザイで大喜び／ピースサイン）。
髪型○○・服○○まで描いた完成形でOK。背景は純白(#FFFFFF)。
```

### まず作る最小セット
- 妹ベース／姉ベース（各1）
- 各キャラの髪（まず1〜2種ずつ）
- 各キャラのコーデ（まず2〜3種ずつ：例 ピンクワンピ／セーラー服／パーカー）
→ 白背景のまま Claude に渡す → 切り抜き → 実装へ。

## 9. Claude の切り抜き方法（再現用メモ）
- **コーデ（着せた体）**：画像の四隅から flood-fill で"外側の白だけ"透明化（服の白い襟・袖は残る）。
- **髪パーツ**：純白（高明度・低彩度）を全体的に透明化（髪に白はないので安全。穴も抜ける）。
- 仕上げに縁のアンチエイリアス処理＋縮小・圧縮（軽量化）。
- ※将来はこの処理を小スクリプト化して Claude Code 側に置いてもよい。
