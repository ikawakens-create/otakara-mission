# AVATAR_SPEC.md — アバター着せ替えシステム 確定仕様（v4・確定版）

おたからミッションのアバターの**正の仕様**。このファイル単体で誰でも再開できる。
端末前提：**Android／保存アセット 768×1024px・WebP**。

> v4（確定版）：v3のレビュー残課題を解消 —
> 正規化は「均一リサイズが基本／検出は補正用」と明確化＋フォールバック連鎖、特別立ち絵の扱いを確定、
> テンプレ数値の算術を整合、リアクションを本仕様の対象外と線引き、バックアップ対象を明記。

---

## 0. 用語
- **ベース**：そのキャラの「はげ・下着・正面直立」の基準画像。全パーツはこのベースに合わせて作る/正規化する。
- **コーデ（outfit）**：体＋顔＋服を1枚にした"着せ替え済みボディ"。顔はこの層に含む。
- **髪（hair）**：髪のみ（顔の穴あき）。前髪あり/なしは別の髪型として持つ。

---

## 1. 確定した方式（コア）
1-1. **レイヤー透過画像方式**（手描きSVG/絵文字は不採用）。絵は **Gemini生成**、Claudeは「指示書／切り抜き・正規化／実装」担当。
1-2. パイプライン：`Gemini白背景生成 → Claudeが透過＋規格へ正規化(リサイズ/位置合わせ/圧縮) → アプリへ`。
1-3. **全パーツは"そのキャラのベースに合わせて"生成する**（co-registration）。これが正規化成立の大前提。
1-4. 基本立ち絵2種（姉用・妹用／別の顔・別の体・両方とも正面直立）＋特別立ち絵（まるごと1枚）。
1-5. 髪・服は姉用/妹用を別々に用意（体が別）。服は"編集モード"で出す。
1-6. アバターをホーム/ミッション/ガチャに登場。感情は動き＋キラ＋吹き出し＋特別立ち絵（**表情差分はやらない**）。
1-7. 軽量化はClaude側で実装時に行う。

---

## 2. 規格・数値（確定）

### 2-1. 保存アセット共通
- 形式：**WebP（透過）**。元PNGは別管理可。
- 寸法：**768 × 1024 px（3:4）固定**。全アセット必ずこの寸法に正規化。
- 容量目標：**1枚 ≤ 70KB**、初期セット合計 **≤ 4MB**。
- ピッカー用サムネ：**192 × 256 px・≤ 12KB**（一覧で原寸を読まない）。

### 2-2. キャラ別ベーステンプレート（768×1024座標）★身長差はここで表現
固定座標は**キャラごと**。図形の上下幅（extent）と頭身で差を出す。**値は目安（実測で±数%微調整可）**。

| 項目 | 妹（younger） | 姉（sister） |
|---|---|---|
| 中心線 x | 384 | 384 |
| 頭頂 y | 120 | 80 |
| あご y（≒頭頂+頭高） | 420 | 360 |
| 足裏 y | 980 | 1010 |
| 頭の高さ（不変量） | 300 | 280 |
| 最大横幅 | ≤ 600 | ≤ 540 |
| 頭身（参考） | 約2.9頭身・まるい | 約3.3頭身・ほっそり |

算術確認：妹 120+300=420（あご）、全身120–980=860→2.9頭身。姉 80+280=360、全身80–1010=930→3.3頭身。整合OK。

### 2-3. 微調整（親「位置調整モード」）
- offsetX/Y：既定 0、範囲 **−80〜+80 px**／scale：既定 1.0、範囲 **0.85〜1.15**。確定値は data に焼き込む。

### 2-4. 描画・性能
- アバター枠 **3:4 固定**、各層を同一枠に**絶対配置**（contain）。
- 同時レイヤー最大6（bg/outfit/hair/hat/accessory/pet）＋special override。
- 表示中アバターは**プリロード**。ピッカーは**サムネ**＋遅延読み込み。

## 3. 正規化アルゴリズム（確定）★基本は均一リサイズ・検出は補正用

**大原則**：全パーツは§11どおり「ベースと同じ枠・同じ頭位置」で生成されている（co-registered）。
よって正規化は**原則「768×1024へ縦横比保持の均一リサイズ」だけで揃う**。下の検出は"ズレていた時の補正"に使う補助であり、必須の高精度処理ではない。

**補正の基準点（カテゴリ別／合わせ先は§2-2テンプレ）**：

| category | 基準点 |
|---|---|
| outfit | 不透明領域の**上端＝頭頂**をテンプレ頭頂yへ。頭の高さをテンプレ頭高に合わせ拡縮（顔サイズ不変＝③対策）。 |
| hair | 透明な**「顔の穴」の上端＝頭頂**をテンプレ頭頂yへ。穴幅を頭幅基準で拡縮。 |
| hat | 不透明領域の**下端＝頭頂**へ。頭幅基準で拡縮。 |
| accessory | 指定アンカー（顔横/手元）に offset 配置。固定スケール。 |
| pet | **足裏ライン**基準で配置。 |
| background | 全面ベタ（登録不要・§5-2例外）。 |
| special | 全身が枠に収まる uniform fit のみ（部位登録なし）。 |

**処理手順（Claude）**
1. 背景透過（outfit/special＝四隅flood-fillで外側白のみ／hair等＝純白を全体透明化）。
2. **均一リサイズで768×1024へ**（基本はこれで一致）。
3. ズレが見える場合のみ上表の基準点で補正移動/拡縮。
4. **フォールバック連鎖**：自動補正で合わなければ → 親「位置調整モード」で手動 → offset/scale を data に焼き込む（最終手段で必ず直せる）。
5. WebP化（容量目標まで）＋サムネ生成。
※この処理は将来スクリプト化し Claude Code 側に置いてよい。

## 4. 初期状態・既定装備（確定）
- **はげ/下着の素の状態はどの画面でも表示しない。**
- 各キャラに **starter のコーデ＋髪を必ず装備**。`AvatarConfig` 既定＝starter。
- **outfit と hair は常時装備（"なし"不可）**。hat/accessory/pet/background は"なし"可。
- starter は owner ごとに最低 outfit×1・hair×1 を `starter:true` で定義。

---

## 5. 命名・配置・データ規約
### 5-1. id / フォルダ
- フォルダ：`src/assets/avatar/<owner>/<category>/<id>.webp`、サムネ `.../<id>.thumb.webp`
- id：`<cat>_<owner>_<name>`（例 `out_younger_pinkdress` / `hair_sister_bob` / `bg_both_sky` / `sp_younger_banzai`）
- cat略号：out / hair / hat / acc / pet / bg / sp
### 5-2. 例外
- background は全面ベタ＝「端に触れない」条件の対象外。
- special は全身まるごと＝部位登録の対象外（768×1024/3:4は守る）。
### 5-3. データ保護（厳守・再掲）
- **配布済み id は削除・改名しない（追加のみ）**。マイグレーションは加算のみ／読み込み失敗時に既存を上書きしない。

## 6. データ構造（実装用）
```ts
type AvatarAsset = {
  id: string;
  category: "outfit"|"hair"|"hat"|"accessory"|"pet"|"background"|"special";
  owner: "sister"|"younger"|"both";
  label: string;            // ひらがな名
  imagePath: string;        // 768×1024 WebP
  thumbPath: string;        // 192×256 WebP
  rarity?: Rarity;
  offsetX?: number;         // 既定0（−80〜+80）
  offsetY?: number;
  scale?: number;           // 既定1.0（0.85〜1.15）
  starter?: boolean;
};
type AvatarConfig = {       // profile内・既存を拡張
  outfitId: string; hairId: string;            // 必須（常に装備）
  hatId?: string; accessoryId?: string; petId?: string; backgroundId?: string;
  specialId?: string;       // セット時は特別立ち絵をまるごと表示（レイヤー無視）
};
```
- 所持は既存 `ownedItemIds` を流用（アバター素材idを格納）。

---

## 7. 特別立ち絵（special）の扱い（確定）★中2解消
- **所持**：他素材と同じく `ownedItemIds` に id を持つ。ガチャ/コンプ報酬での付与＝id を ownedItemIds に追加。
- **装備**：`specialId` をセットすると**全レイヤーを上書き**して1枚表示（優先度 specialId > レイヤー）。
- **UX**：着せ替え画面に「とくべつ」枠を設け、未所持はロック表示。「とくべつをはずす」で通常の重ね着せ替えに戻る（outfit/hairの装備は保持）。
- **owner絞り込み**：§8と同様、自分の special と owner=="both" のみ表示。

---

## 8. owner ↔ profile 絞り込み（確定）
- 見られる/装備できるのは **owner==自分 または "both"** のみ。着せ替え・図鑑・ガチャ付与すべてに適用。

- 妹：元気・にっこり全開・まるっと小さめ／テーマ色ピンク／頭身まるめ・背低め。

## 11. 素材づくり（Gemini 並行・要点）
- 共通：ちび約3頭身・全身正面直立・縦長3:4・中央・全周に余白・純白背景・影/模様なし・線画くっきり。**必ずベース画像を添付して同じ枠・同じ頭位置に合わせる（co-registration）。**
- ベース：はげ＋白ノースリーブ＋白短パン＋はだし、髪なし。妹→（妹を添付して）姉。
- 髪：髪のみ・顔の穴あき。姉用/妹用を別々に。
- コーデ：ベースを"編集"し服だけ変更（顔/ポーズ/位置/大きさ不変）。
- 特別立ち絵：まるごと完成形。
- 詳細プロンプトは別冊 `GEMINI_ASSET_PROMPTS.md`。
- 素材は本人のAI生成（私的・家庭内利用）。

## 12. バックアップ／エクスポートの対象（確定）★軽5解消
- **画像アセットはアプリ同梱（リポジトリ管理）＝端末バックアップの対象外**。アプリ更新で常に再取得される。
- バックアップ/エクスポートは**セーブデータのみ**（所持id・装備(AvatarConfig)・ポイント・かけら・進捗）。
- 復元時：所持id・装備を戻せば、画像は同梱アセットから再描画される。

## 13. 本仕様の対象外（線引き・確定）★軽4解消
- **リアクション演出（動き・吹き出し秒数）は本仕様＝3-Bの対象外。別PR（登場＆リアクション）で扱う。**
  - 参考の暫定値：ぴょこ0.3s／吹き出し1.5s／全クリア1.8s（確定ではない）。
