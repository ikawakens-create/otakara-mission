# STEP_1_5_SPEC.md — 週間・月間ビュー＋やる気アップ機能

ステップ1（日替わり）とステップ2（ガチャ）の間に差し込む追加ステップ。
データは既存の dailyRecords（YYYY-MM-DD単位）から計算するので、新しい記録の仕組みは増やさず、
主に「見せ方」と「リカバリー」「段階ごほうび」を足す。

## このステップに入れるもの / 見送るもの

入れる：
- 週間ビュー（紙版の曜日×ミッション表）
- 月間ビュー（カレンダーで達成日を色分け）
- ミッション別達成率（苦手項目の可視化）
- リカバリーミッション（「あと1個」の日を後から救済）
- 月間ごほうび連動（日→週→月で段階化）
- 上記をおうちの人モードで設定できる項目

今回は見送る（後のステップで追加可）：
- 連続記録（ストリーク）

## 1. 画面構成

トップは今のまま「きょう」。上部に切り替えタブを追加する： [ きょう ] [ こんしゅう ] [ こんげつ ]
普段は「きょう」。週末や月末に「こんしゅう/こんげつ」を見る運用。子どもは普段「きょう」だけでOK。

## 2. 週間ビュー（こんしゅう）

- レイアウト：縦にミッション、横に曜日（月〜日）。紙版の表と同じ並び。
- 各セル：その日そのミッションにスタンプ済みなら表示、未なら空。過去日は編集不可（閲覧のみ）。
- 日ごとの状態を列の下に表示：コンプリート＝金の花丸 / リカバリーで達成＝銀の花丸 / 未達＝なし
- 週サマリー：今週コンプリート日数（例 4/7）、週間ボーナス（5日）までの進捗メーター（あと◯日）
- 前の週／次の週へ移動できる（weekStartDate基準で計算）。
- ミッション別達成率（今週分）を各行末または下部に表示。達成率が低い行に「にがて」マークを付ける。

## 3. 月間ビュー（こんげつ）

- カレンダー：その月の各日をマス目で表示。
- 各日の色分け：金＝コンプリート / 銀＝リカバリー達成 / うすい色＝一部達成 / 白＝未達 / 灰＝未来の日
- 日をタップ → その日の詳細（どのミッションができたか）を表示。
- 月サマリー：今月のコンプリート日数、リカバリー回数、月間ごほうびまでの進捗。
- 前の月／次の月へ移動できる。ミッション別達成率（今月分）も表示。

## 4. リカバリーミッション

目的：「あと1個で全部だった惜しい日」を、追加の頑張りで取り返せるようにして、やる気を折らない。

対象になる日（厳格に限定）：
- 過去の日で、全ミッション中ちょうど1個だけ未達の日のみ（＝「あと1個」の日）。
- 半分しかやっていない日などは対象外。当日や未来の日も対象外。

回数制限（ズル防止）：
- 1週間に既定2回まで（おうちの人モードで 1〜3 に変更可）。
- 使用回数は WeekState.recoveryUsedThisWeek で管理し、週またぎでリセット。

やり方：
- 週間/月間ビューで「あと1個」の日にリカバリーチャンスの印を出す。
- タップ → リカバリー専用ミッション一覧（おうちの人が設定）から1つ選んでクリア。
- 成立すると、その日をコンプリート扱い（銀の花丸）にする。

効果（設定で調整可。下記がデフォルト）：
- その日をコンプリート扱いにし、週の達成日数にカウントする（週間ボーナスに前進できる）。
- ボーナスポイント +1。
- ガチャも1回引ける（recoveryGrantsGacha のデフォルトは true）。
- 見た目は通常コンプリート（金）と必ず区別する（銀）。

リカバリー専用ミッションの例（おうちの人が編集）：
絵本を1さつ よむ / おもちゃを 10こ かたづける / おてつだいを もう1つ する など。

## 5. 月間ごほうび連動（段階化）

短期（日）：その日コンプリート → +1pt ＆ ガチャ1回（ステップ2で実装）
中期（週）：週5日コンプリート → 週間ボーナス（ステップ4で本実装、本ステップでは進捗表示まで）
長期（月）：月◯日コンプリート（既定20日／設定可） → 月間スペシャル報酬

- 月間スペシャル報酬：ボーナスポイントまとめ付与＋スペシャルガチャ券1枚を想定。
  ガチャ未実装の段階では、ポイント付与と「券を1枚もらった」記録だけ先に作る。
- 同じ月に二重付与しないよう monthlyRewardGiven["YYYY-MM"] で管理。
- しきい値・報酬量はおうちの人モードで変更可。

## 6. おうちの人モードに追加する設定

- リカバリー専用ミッションの一覧（追加/削除/編集）。
- リカバリーの週あたり回数上限（1〜3、既定2）。
- 「リカバリーでもガチャを引ける」ON/OFF（既定ON）。
- 月間ごほうびのしきい値（既定20日）と報酬量。

## 7. データ構造の追記

DailyRecord に追加：
- `recovered?: boolean` — リカバリーで達成扱いにした日
- `recoveryMissionId?: string` — 使ったリカバリーミッション

WeekState に追加：
- `recoveryUsedThisWeek: number` — 今週のリカバリー使用回数

Profile に追加：
- `monthlyRewardGiven: Record<string, boolean>` — キー "YYYY-MM"
- `specialGachaTickets: number` — スペシャルガチャ券の枚数

AppSettings に追加：
- `recoveryMissions: RecoveryMissionDef[]`
- `recoveryWeeklyLimit: number` — 既定2
- `recoveryGrantsGacha: boolean` — 既定true
- `monthlyGoalDays: number` — 既定20
- `monthlyRewardPoints: number` — 既定10

## 8. 完了条件（AC）

- 「きょう/こんしゅう/こんげつ」を切り替えられる。
- 週間ビューが紙版同様の曜日×ミッション表で、今週の達成状況とミッション別達成率を表示する。
- 月間ビューがカレンダーで、コンプリート/リカバリー/一部/未達を色分け表示する。
- 「あと1個」の過去日にのみリカバリーが出て、週上限を超えると使えない。
- リカバリー成立で当該日が銀の花丸＝コンプリート扱いになり、週の達成日数に加算される。通常達成（金）と見た目が区別される。
- 月◯日達成で月間報酬が一度だけ付与され、同月に二重付与されない。
- おうちの人モードでリカバリーミッション・各しきい値を編集でき、子どもは触れない。
- 既存データ（ステップ1で保存済みの記録）が壊れずに引き継がれる（マイグレーション）。
