# STEP_2B_34C.md — §3-4c 激アツ専用ビート「出てこない→もう1回→確定」

## 目的
最高レア（ダイヤ＝確定）のときだけ、特別な激アツ演出を入れる：
**ハンドルを回す→でも何も落ちてこない（あれ…？）→もう1回！→ドンッと派手なカプセル＋かくてい！！**。
「出てこない＝絶対いいやつ」という、子どもが安心して大興奮できる合図にする。結果は変えない（演出だけ）。

## 子ども向けの絶対ルール
- この激アツビートは **`cutin === "confirmed"`（＝ダイヤ）のときだけ**出す。
- **ガセ・通常では絶対に出さない**（出てこない＝期待外れ、を子どもにさせない）。

---

## 1. 発生条件（Gacha.tsx）
- `GachaScreen` に **`forcedRetry?: boolean`**（テスト用）を追加。
- 引いた時に判定：
  ```ts
  const retry = forcedRetry ?? (cut === "confirmed");  // cut は決定済みカットイン
  setIsRetry(retry);
  ```
  （`isRetry` を state に持つ。`cut` は §CUTIN で決めた値）

## 2. capsule フェーズに前段を差し込む（自動再生）
capsule フェーズに、retry のときだけ**前段ステージ**を入れる。タップは増やさず**自動で進む**。

`retryStage` を state に追加：`"none" | "shaking" | "again" | "drop"`

- **非retry時**：従来どおり。capsule に入ったら即「`drop`」相当（ハンドル回転＋カプセル落下＋カットイン）。
  → **既存挙動を変えないこと（回帰なし）**。
- **retry時**：capsule に入ったら次の順に**自動進行**（useEffect＋タイマー、クリーンアップ必須）：
  1. `shaking`（約1.1秒）：`turning=true`／`dropCapsule=null`（**何も落ちない**）／文言「**あれ…？ でてこない！？**」。カットインはまだ出さない。
  2. `again`（約0.9秒）：`turning=true`（もう一度回る・できれば少し激しめ）／`dropCapsule=null`／文言「**もう1回！**」。
  3. `drop`：`dropCapsule` を出して**派手なカプセルが落ちる**＋**かくてい！！カットイン**を表示。
- `drop` に入ったら、あとは従来どおり（タップで open→reveal、reveal は burst で大フラッシュ）。

### 実装メモ
- 前段の自動進行は **useEffect（isRetry かつ animPhase==="capsule" で起動）＋ setTimeout**。
  フェーズが変わる/スキップ/アンマウント時に **clearTimeout でクリーンアップ**する（タイマー残り対策）。
- 「とばす ▶」スキップは従来どおり効くこと（押したらタイマーを止めて result へ）。
- カットイン（confirmed）は retry時は **`drop` ステージで**出す（shaking/again では出さない）。
  非retry時は従来どおり capsule 入場時に出す。
- 文言「あれ…？ でてこない！？」「もう1回！」は定数化（後で変えやすく）。
- （任意）`again` の回転を少し速く/激しくするCSSクラスを足してもよい。なくても可。

## 3. テスト（TestGacha）
- **「🔥 でてこない演出」トグル/ボタン**を追加し、`forcedRetry={true}` で渡す。
- forcedRetry のときは、選んだレア度に関わらず retry の流れ＋かくてい！！を再生（テスト用）。
  - 実装が楽なら、forcedRetry 時はカットインも confirmed 相当で表示してよい。

## 完了条件（AC）
- ダイヤ（confirmed）のとき、capsule で「**回す→出てこない（あれ…？）→もう1回！→ドンッと落ちる＋かくてい！！**」が自動で流れる。
- その後 reveal で大フラッシュ（burst）で祝う。
- **ガセ・通常では絶対に出ない**（confirmed以外は従来どおり）。
- 非retryの通常ガチャ・各シナリオが**従来どおり**（回帰なし）。
- 「とばす ▶」でいつでも結果へ飛べる（タイマーが残らない）。
- 演出テストの「🔥 でてこない演出」で、この激アツを狙って確認できる。
- 結果（レア度・景品）は不変。`npm run build` が通る。
